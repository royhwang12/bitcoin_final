# bitcoin_final

Computer Networks final project: a small peer-to-peer proof-of-work blockchain with a local dashboard.

## Submitted Documents

- `README.md`: code structure, setup, compilation/build notes, and usage.
- `Design.md`: blockchain design, peer-to-peer protocol, demo application design, and limitations.
- `TESTING.md`: test strategy, commands, checkpoint list, and observed results.

## Code Structure

```text
bitcoin_final/
  README.md
  Design.md
  TESTING.md
  pyproject.toml
  uv.lock

  blockchain/
    __init__.py
    block.py          # Block model, genesis block, header hashing, serialization
    chain.py          # Chain state, transaction/block validation, append, fork choice
    pow.py            # Proof-of-work difficulty checks and mining helpers
    tx.py             # Transaction model, signing payloads, transaction hashing
    wallet.py         # ECDSA wallet creation, signing, public-key verification

  net/
    __init__.py
    messages.py       # Length-prefixed JSON message framing and message constants
    peer.py           # Peer runtime, mempool, mining loop, gossip, chain sync
    tracker.py        # Tracker runtime for peer registration and peer-list updates

  dashboard/
    __init__.py
    app.py            # FastAPI app and `blockchain-dashboard` entry point
    orchestrator.py   # In-process controller for tracker, peers, wallets, txs
    static/index.html # Plain HTML/CSS/JS browser dashboard

  tests/
    test_chain.py
    integration_demo.py
```

## Requirements

- Python `3.10+`
- [`uv`](https://docs.astral.sh/uv/)

## Setup and Dependencies

```bash
uv sync
```

This creates/updates the project virtual environment and installs runtime dependencies from `pyproject.toml` and lockfile constraints from `uv.lock`.

Make sure to install uv first
Mac: brew install uv

## Compilation / Build

This is a Python-only project, so there is no separate compile step. Dependency resolution and editable package installation are handled by:

```bash
uv sync
```

`pyproject.toml` uses `setuptools` as the build backend and defines the console script:

```text
blockchain-dashboard = "dashboard.app:main"
```

## Usage

### Dashboard Demo

```bash
uv run blockchain-dashboard
```

Then open `http://127.0.0.1:8000`.

From the UI you can:

- Start the tracker
- Create wallets (with optional initial balances)
- Create peers (optionally start mining)
- Submit transactions
- Watch live chain/mempool/balance state updates

### Direct Module Entrypoints

The tracker and peer can also be started directly as modules:

```bash
uv run python -m net.tracker --host 127.0.0.1 --port 9000
uv run python -m net.peer --port 9201 --tracker-host 127.0.0.1 --tracker-port 9000
```

## Tests

```bash
uv run python -m unittest tests.test_chain or python3 tests/test_chain.py
uv run python tests/integration_demo.py or python3 tests/integration_demo.py

```

See `TESTING.md` for the full checkpoint list and expected outcomes.
